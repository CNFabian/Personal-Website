# Database Schema

## Overview

The current database is SQLite with a single `users` table. The casino expansion requires significant schema additions to support avatars, chip balances, transaction history, game records, friends, and more.

SQLite remains appropriate for the expected scale (< 1,000 active users). If the platform grows significantly, migration to PostgreSQL is straightforward since the schema uses standard SQL features.

---

## Current Schema

```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT UNIQUE NOT NULL COLLATE NOCASE,
    password_hash TEXT,
    wins INTEGER DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

This table currently stores authentication data and a simple win counter. The new schema expands on this while maintaining backward compatibility.

## Expanded Schema

### users (Modified)

```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT UNIQUE NOT NULL COLLATE NOCASE,
    password_hash TEXT,
    email TEXT UNIQUE COLLATE NOCASE,

    -- Currency
    chip_balance INTEGER DEFAULT 1000,

    -- Avatar (stored as JSON blob)
    avatar_data TEXT DEFAULT NULL,

    -- Stats (aggregated for quick access)
    total_wins INTEGER DEFAULT 0,
    total_losses INTEGER DEFAULT 0,
    total_games_played INTEGER DEFAULT 0,

    -- Daily bonus tracking
    last_daily_bonus DATETIME DEFAULT NULL,
    daily_streak INTEGER DEFAULT 0,

    -- Bankruptcy protection
    last_relief_package DATETIME DEFAULT NULL,

    -- Account status
    is_active INTEGER DEFAULT 1,
    is_banned INTEGER DEFAULT 0,
    ban_reason TEXT DEFAULT NULL,

    -- Timestamps
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    last_login DATETIME DEFAULT NULL,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_users_username ON users(username);
CREATE INDEX idx_users_chip_balance ON users(chip_balance);
CREATE INDEX idx_users_total_wins ON users(total_wins);
```

**Migration from current schema:** Add new columns with defaults. Existing users get 1,000 chips, null avatar (prompts creation on next login), and their existing `wins` value copied to `total_wins`.

### game_stats (Per-Game-Type Statistics)

```sql
CREATE TABLE game_stats (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    game_type TEXT NOT NULL,  -- 'ratscrew', 'gin_rummy', 'holdem_hu', 'holdem_multi'

    wins INTEGER DEFAULT 0,
    losses INTEGER DEFAULT 0,
    draws INTEGER DEFAULT 0,
    games_played INTEGER DEFAULT 0,

    -- Game-specific stats (JSON for flexibility)
    extra_stats TEXT DEFAULT '{}',
    -- Examples:
    --   ratscrew: { "totalSlaps": 150, "correctSlaps": 120, "slapAccuracy": 0.80 }
    --   gin_rummy: { "totalGins": 12, "totalKnocks": 45, "undercuts": 8 }
    --   holdem: { "handsWon": 200, "biggestPot": 5000, "allInWins": 15 }

    -- Chip stats for this game type
    total_chips_won INTEGER DEFAULT 0,
    total_chips_lost INTEGER DEFAULT 0,
    biggest_single_win INTEGER DEFAULT 0,

    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,

    FOREIGN KEY (user_id) REFERENCES users(id),
    UNIQUE(user_id, game_type)
);

CREATE INDEX idx_game_stats_user ON game_stats(user_id);
CREATE INDEX idx_game_stats_type ON game_stats(game_type);
```

### game_sessions (Individual Game Records)

```sql
CREATE TABLE game_sessions (
    id TEXT PRIMARY KEY,  -- UUID generated by server
    game_type TEXT NOT NULL,

    -- Room info
    room_code TEXT,
    table_id TEXT,  -- which lobby table this was played at

    -- Wager info
    wager_amount INTEGER DEFAULT 0,
    pot_total INTEGER DEFAULT 0,

    -- Result
    winner_id INTEGER,
    status TEXT DEFAULT 'in_progress',  -- 'in_progress', 'completed', 'forfeit', 'cancelled'

    -- Game-specific result data (JSON)
    result_data TEXT DEFAULT '{}',
    -- Examples:
    --   ratscrew: { "finalScoreP1": 52, "finalScoreP2": 0, "totalSlaps": 25, "duration_seconds": 180 }
    --   gin_rummy: { "rounds": 5, "finalScoreP1": 105, "finalScoreP2": 87 }
    --   holdem: { "handsPlayed": 30, "finalStackP1": 1500, "finalStackP2": 500 }

    -- Timestamps
    started_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    ended_at DATETIME DEFAULT NULL,

    FOREIGN KEY (winner_id) REFERENCES users(id)
);

CREATE INDEX idx_game_sessions_type ON game_sessions(game_type);
CREATE INDEX idx_game_sessions_status ON game_sessions(status);
CREATE INDEX idx_game_sessions_started ON game_sessions(started_at);
```

### game_session_players (Players in Each Game)

```sql
CREATE TABLE game_session_players (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    game_session_id TEXT NOT NULL,
    user_id INTEGER NOT NULL,

    seat_number INTEGER,  -- 1-6 depending on game
    is_winner INTEGER DEFAULT 0,

    -- Chip movement for this player in this game
    chips_wagered INTEGER DEFAULT 0,
    chips_won INTEGER DEFAULT 0,
    net_result INTEGER DEFAULT 0,  -- chips_won - chips_wagered

    -- Player state at end of game
    final_state TEXT DEFAULT '{}',  -- JSON, game-specific

    joined_at DATETIME DEFAULT CURRENT_TIMESTAMP,

    FOREIGN KEY (game_session_id) REFERENCES game_sessions(id),
    FOREIGN KEY (user_id) REFERENCES users(id)
);

CREATE INDEX idx_gsp_session ON game_session_players(game_session_id);
CREATE INDEX idx_gsp_user ON game_session_players(user_id);
```

### transactions (Chip Ledger)

```sql
CREATE TABLE transactions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,

    type TEXT NOT NULL,
    -- Types: 'game_win', 'game_loss', 'wager_escrow', 'wager_return',
    --        'daily_bonus', 'signup_bonus', 'relief_package',
    --        'achievement', 'cosmetic_purchase',
    --        'deposit', 'withdrawal' (future real-money)

    amount INTEGER NOT NULL,  -- positive = credit, negative = debit
    balance_after INTEGER NOT NULL,

    -- Related entities (nullable)
    game_session_id TEXT DEFAULT NULL,
    related_user_id INTEGER DEFAULT NULL,

    description TEXT,

    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,

    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (game_session_id) REFERENCES game_sessions(id),
    FOREIGN KEY (related_user_id) REFERENCES users(id)
);

CREATE INDEX idx_transactions_user ON transactions(user_id);
CREATE INDEX idx_transactions_type ON transactions(type);
CREATE INDEX idx_transactions_created ON transactions(created_at);
```

### friends

```sql
CREATE TABLE friends (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    requester_id INTEGER NOT NULL,
    receiver_id INTEGER NOT NULL,

    status TEXT DEFAULT 'pending',  -- 'pending', 'accepted', 'blocked'

    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,

    FOREIGN KEY (requester_id) REFERENCES users(id),
    FOREIGN KEY (receiver_id) REFERENCES users(id),
    UNIQUE(requester_id, receiver_id)
);

CREATE INDEX idx_friends_requester ON friends(requester_id);
CREATE INDEX idx_friends_receiver ON friends(receiver_id);
CREATE INDEX idx_friends_status ON friends(status);
```

### achievements

```sql
CREATE TABLE achievements (
    id TEXT PRIMARY KEY,  -- 'first_win', 'win_streak_10', 'high_roller', etc.
    name TEXT NOT NULL,
    description TEXT NOT NULL,
    category TEXT NOT NULL,  -- 'general', 'ratscrew', 'gin_rummy', 'holdem', 'social'
    chip_reward INTEGER DEFAULT 0,
    icon TEXT DEFAULT NULL,  -- sprite/icon reference
    rarity TEXT DEFAULT 'common'  -- 'common', 'uncommon', 'rare', 'epic', 'legendary'
);

CREATE TABLE user_achievements (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    achievement_id TEXT NOT NULL,

    unlocked_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    reward_claimed INTEGER DEFAULT 0,

    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (achievement_id) REFERENCES achievements(id),
    UNIQUE(user_id, achievement_id)
);

CREATE INDEX idx_user_achievements_user ON user_achievements(user_id);
```

## Migration Strategy

### Step 1: Non-Breaking Additions
Add new columns to the `users` table with defaults. No existing functionality breaks.

```sql
ALTER TABLE users ADD COLUMN chip_balance INTEGER DEFAULT 1000;
ALTER TABLE users ADD COLUMN avatar_data TEXT DEFAULT NULL;
ALTER TABLE users ADD COLUMN total_losses INTEGER DEFAULT 0;
ALTER TABLE users ADD COLUMN total_games_played INTEGER DEFAULT 0;
ALTER TABLE users ADD COLUMN email TEXT DEFAULT NULL;
ALTER TABLE users ADD COLUMN last_daily_bonus DATETIME DEFAULT NULL;
ALTER TABLE users ADD COLUMN daily_streak INTEGER DEFAULT 0;
ALTER TABLE users ADD COLUMN last_relief_package DATETIME DEFAULT NULL;
ALTER TABLE users ADD COLUMN is_active INTEGER DEFAULT 1;
ALTER TABLE users ADD COLUMN is_banned INTEGER DEFAULT 0;
ALTER TABLE users ADD COLUMN ban_reason TEXT DEFAULT NULL;
ALTER TABLE users ADD COLUMN last_login DATETIME DEFAULT NULL;
ALTER TABLE users ADD COLUMN updated_at DATETIME DEFAULT CURRENT_TIMESTAMP;
```

### Step 2: Rename for Consistency
```sql
-- Copy wins → total_wins (SQLite doesn't support RENAME COLUMN in older versions)
-- For newer SQLite (3.25+):
ALTER TABLE users RENAME COLUMN wins TO total_wins;
```

### Step 3: Create New Tables
Run the CREATE TABLE statements above for all new tables.

### Step 4: Seed Data
Insert achievement definitions. Existing users get their `chip_balance` set to 1000 and `total_wins` populated from the old `wins` column.

### Step 5: Update Server Code
Modify `db.js` to include new query functions for all new tables. Add migration versioning to track which migrations have been applied.

## Database Query Patterns

### Frequently Accessed (Cache-Worthy)

- `getUserById(id)` — Returns user profile including avatar and chip balance. Called on every login, lobby join, and game start.
- `getLeaderboard(gameType, limit)` — Top players by wins or chips. Called when viewing the leaderboard wall.
- `getChipBalance(userId)` — Called before every wager to validate funds.

### Write-Heavy

- `updateChipBalance(userId, delta)` — Called on every game result, daily bonus, and achievement. Must be atomic (use transactions).
- `logTransaction(...)` — Called alongside every chip balance change.
- `updateGameStats(userId, gameType, result)` — Called at end of every game.

### Periodic

- `claimDailyBonus(userId)` — Called once per day per user.
- `getTransactionHistory(userId, limit, offset)` — Called when viewing chip history at the cashier.
- `getFriendsList(userId)` — Called on lobby join to show online friends.
